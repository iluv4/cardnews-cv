# 제품 비즈니스 케이스 — Korean Card-News (v2, 하이브리드)

> CV로 시작한 카드뉴스 엔진을 제품으로 전환하는 판단 자료.
> **v1 폐기 사유:** v1은 "LLM은 한글이 깨진다·AI 티 난다·비용"을 3대 해자로 삼았으나,
> **2026-04 GPT-5.5 + GPT Image 2/2.5 출시로 이 전제들이 대부분 무효화**됐다(아래).
> v2는 그 현실 위에서 "픽셀 생성은 위임, CV 자산은 컨트롤·검색·QA로 재배치"하는 B 방향.

## 한 줄 요약
**픽셀은 GPT Image 2가 그린다. 우리는 그 위에서 "어떤 레퍼런스를, 어떤 레이아웃으로,
브랜드에 정확히 맞게" 지시하고 검수하는 층이 된다.** 학습된 디텍터 + 레퍼런스 검색은
GPT가 못 가진 자산이고, 픽셀을 직접 그리는 싸움은 더 이상 이길 수 없다.

---

## 무엇이 바뀌었나 (전제 재설정)
- GPT Image 2/2.5는 **한글 포함 CJK 인-이미지 텍스트를 거의 무오류로 렌더**한다.
  포스터·인포그래픽·슬라이드 등 "텍스트+구조" 포맷을 정조준해 출시됨.
- 장당 비용도 작다(1024² 고품질 ~$0.21, 중품질 ~$0.05, Batch -50%) → 10장 덱 수백 원대.

| v1이 내세운 해자 | 현재 |
|---|---|
| 한글 가독성 | ❌ 죽음 (GPT가 해결) |
| AI 티 없음 / 일관성 | ⚠️ 약화 (GPT가 빠르게 따라옴) |
| 장당 비용 0 | 🔸 우위는 있으나 절대액이 작아 결정타 아님 |

→ **"우리가 픽셀을 더 잘 그린다"는 더는 사업 근거가 아니다.**

## 그래도 GPT가 구조적으로 못 가진 것 (= 남은 방어선)
1. **브랜드 픽셀 정확성** — 정확한 컬러 HEX·지정 폰트·로고 위치/여백. 래스터 생성은
   "거의 맞게" 그릴 뿐 보장 못 한다. 대행사·브랜드팀에겐 이게 합격/불합격을 가른다.
2. **편집 가능한 구조적 출력** — 텍스트 사후 수정·리플로우, 로고 교체, 리사이즈.
   GPT 출력은 납작한 PNG.
3. **레퍼런스 기반 레이아웃 지능** — "이 브랜드가 좋아하는 실제 레이아웃을 찾아
   그대로 따른다"는, 우리 학습 데이터·검색에서만 나오는 자산.

## 우리 자산을 어디에 재배치하나 (B 구조)
픽셀 생성을 GPT Image 2에 넘기고, 기존 자산을 그 **앞단(컨트롤)** 과 **뒷단(QA)** 에 둔다:

```
[사용자 텍스트 + 브랜드킷]
      │
      ▼
(1) reflib/ 검색  ──▶  브랜드/주제에 맞는 "좋은 실제 레퍼런스" 후보 제시 (CLIP)
      │
      ▼
(2) 디텍터(best.pt) ──▶  그 레퍼런스에서 정확한 레이아웃(블록 위치/계층) 추출
      │
      ▼
(3) 레이아웃→지시 변환 ──▶  GPT Image 2에 줄 정밀 프롬프트/컨트롤(영역·텍스트·톤)
      │
      ▼
   GPT Image 2  ──▶  픽셀 렌더 (한글 OK)
      │
      ▼
(4) 디텍터로 출력 재검수 ──▶  브랜드/레이아웃 규칙 위반 자동 QA → 재생성 루프
      │
      ▼
(5) 결정론적 오버레이(선택) ──▶  로고·핵심 텍스트는 우리 벡터 엔진으로 픽셀-정확 덮어쓰기
```

- (1)(2)는 **GPT가 못 하는 부분** — 우리 학습 디텍터(mAP@50-95=0.718)와 레퍼런스 라이브러리.
- (4) 디텍터를 **생성기가 아니라 평가기(QA)** 로 쓰는 게 핵심 전환 — "잘 그렸나"를 자동 채점.
- (5)는 방어선 1·2를 살리는 선택지: 브랜드 결정 요소만 벡터로 덮어 정확성·편집성 확보.

## 이미 가진 것 / 새로 필요한 것
- **그대로 쓰는 자산:** `results/best_model/best.pt`(디텍터), `reflib/`(검색), `service/library/templates.json`(레이아웃 추출 결과), 렌더 엔진.
- **버리는 것:** DS-GAN 픽셀 생성 파이프라인 전체, "우리가 더 잘 그린다" 서사. (논문 베이스라인으로만)
- **새로 만들 것(작음):** GPT Image 2 호출 래퍼 + 레이아웃→프롬프트 변환(3) + 출력 QA 루프(4).
- **모델 학습 불필요.** 추가 비용은 GPT 이미지 API 사용량뿐(덱당 수백 원대).

## Go-to-market (좁게, 방어선이 통하는 곳)
대중 카드뉴스 생성은 GPT가 먹는다. 우리는 **GPT가 못 만족시키는 좁은 곳**으로:
- **타깃:** 브랜드 일관성이 합격 기준인 곳 — 마케팅 대행사, 공공기관, 프랜차이즈 본사.
- **가치 제안:** "GPT로 빠르게 뽑되, 로고·컬러·폰트는 브랜드 가이드대로 픽셀-정확하게,
  그리고 나중에 수정 가능하게." — 순수 GPT가 못 주는 정확성·편집성.

## 다음 단계 (저위험, 즉시 검증)
1. **B 고리 1장 프로토타입** — reflib 검색 → 디텍터 레이아웃 추출 → GPT Image 2 프롬프트
   → 출력 → 디텍터 QA. 한 장이 도는지가 "하이브리드 제품 vs 환상"을 가른다.
2. **(5) 오버레이 가치 검증** — GPT 출력 위에 우리 벡터로 로고/타이틀 픽셀-정확 덮기.
   "정확성·편집성"이 사용자에게 실제 값을 주는지 확인.
3. **대행사 3곳 인터뷰** — "브랜드 정확성 + 편집성"에 돈을 낼지. GTM 확정 게이트.

> 레퍼런스 검색)을 컨트롤·QA 층으로 옮기는 것** — 새 모델 없이 며칠 단위의 재배치다.
